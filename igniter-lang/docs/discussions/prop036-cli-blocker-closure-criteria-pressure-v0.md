[Challenge]

1. **B6 adversarial scanner self-test (C2-P1 B6-2/B6-3) is not in C4-A's
   closure criterion (runtime-pressure lens).** C2-P1 defines two important
   scanner-trustworthiness fixtures:
   - B6-2: a scanner fixture with an injected bare `present_verified` should
     return FAIL, proving the scanner can detect a forbidden token;
   - B6-3: a scanner fixture with `compiler_profile_source.id_digest_mismatch`
     should be documented as an allowed qualified string.
   C4-A's B6 closure criterion specifies the scan surface table and the "scan
   fails if" conditions, but does not require the adversarial injection fixture.
   A future implementation card is bound to C4-A (the gate), not C2-P1 (the
   track). A correct scanner run over clean outputs could pass C4-A B6 without
   ever proving the scanner's detection capability. This leaves a latent scan-
   trustworthiness gap. Non-blocker for the closure-criteria decision itself; NB
   for B6 executability.

2. **B8-C deferral authority is unspecified.** C4-A B8 item 3 says "or
   explicitly deferred by a named docs/governance card, with the deferral path
   recorded in the closing evidence." C3-P1 says "a named docs/governance card
   explicitly defers source-level visibility." Neither document specifies: who
   must write or sign the deferral card, whether a Research Agent or
   Compiler/Grammar Expert track counts as a "docs/governance card," or whether
   the deferral requires Architect sign-off. A future closure card could claim B8-C
   by pointing to any track doc that mentions deferral, re-introducing self-
   assertion through the deferral path. Non-blocker; NB.

3. **B1's `standalone_artifact_valid: true` self-referential validation.** The
   proof generates, validates, and records validity in its own summary. The C4-A
   B1 criterion requires this summary field but does not name which validation
   path must be invoked (e.g., "the same source validation used in the
   finalization proof and assembler source contract"). C1-P1 specifies this: "the
   file validates with the same source validation used by the finalization
   proof/assembler source contract." C4-A's B1 closure criteria do not reproduce
   that specificity. A future B1 closure card could set `standalone_artifact_valid:
   true` by running a superficial well-formedness check rather than the full
   source-validation chain. Non-blocker given existing proof logic is verified;
   NB for closure-bar precision.
