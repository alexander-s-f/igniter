# History Type Point Access Proof v0

Role: `[Igniter-Lang Research Agent]`
Track: `history-type-point-access-proof-v0`
Status: done
Date: 2026-05-07

## Goal

Build the first executable `History[T]` proof from the planning slice.

This proof is intentionally small:

```text
History[Integer]
  + two synthetic append observations
  + explicit as_of projection
  -> Option[Integer]
```

It proves value-over-time and evidence-linked projection without implementing
the whole temporal system.

## Scope Boundary

[D] This is a proof-local Stage 2 experiment, not a production compiler module.

[D] Parser support for `read job_count_history: History[Integer] from ...` is
not implemented here. The experiment includes `.ig` fixture sketches, then
starts from hand-authored ParsedProgram JSON generated by the runner.

[D] `append` is not source-language mutation. The proof seeds a
`MemoryHistoryBackend` with synthetic append observations before runtime
evaluation.

[D] Option encoding is explicit:

```json
{
  "some": { "kind": "some", "value": "<value>" },
  "none": { "kind": "none" }
}
```

## Positive Fixture

Source sketch:

```text
contract TechnicianJobCountAt {
  input technician_id: String
  input as_of: DateTime

  escape history_read

  read job_count_history: History[Integer]
    from "technicians/{technician_id}/job_count"
    lifecycle :durable

  compute current_count = history_at(job_count_history, as_of)

  output current_count: Option[Integer] lifecycle :session
}
```

Seeded append observations:

```json
[
  {
    "subject": "technicians/tech-synthetic-1/job_count",
    "valid_from": "2026-05-01T00:00:00Z",
    "value": 7,
    "value_type": "Integer"
  },
  {
    "subject": "technicians/tech-synthetic-1/job_count",
    "valid_from": "2026-05-05T00:00:00Z",
    "value": 9,
    "value_type": "Integer"
  }
]
```

Runtime projections:

```text
2026-05-03T10:00:00Z -> { "kind": "some", "value": 7 }
2026-05-06T10:00:00Z -> { "kind": "some", "value": 9 }
```

Each runtime evaluation emits a `history_access_observation` with:

```text
selected_append_ref -> obs/history_append/...
```

## Gaps

[R] Parser gap remains: current parser is not yet the source of truth for
`History[T]` read syntax. Next compiler work should teach parser/goldens the
fixture rather than relying on the hand-authored ParsedProgram.

[R] TypeChecker support is proof-local. Production extraction still needs
generic `TypeRef` resolution for `History[T]`, `Option[T]`, and `DateTime`.

[R] RuntimeMachine support is proof-local. A production path still needs
`temporal_access_node` evaluation and a real TBackend adapter boundary.

[R] Only point access is proven. Range access, aggregates, `BiHistory[T]`,
corrections, streams, and OLAP unification remain out of scope.
