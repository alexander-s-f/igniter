high-value pressure archive, not an implementation backlog.

They converge on one platform thesis:

```text
validated contracts
  + long-lived sessions
  + human/agent review
  + evidence receipts
  + app-local surfaces
  + supervisor-gated research
  -> agent-native application platform
```

What should survive is the value map and graduation discipline. What should not
survive is treating every proposed DSL, UI kit, product app, or runtime concept
as accepted roadmap.

## Values Preserved

- **Evidence before action**: observe, inspect, review, then mutate.
- **Agent as participant**: agents should have explicit state, evidence,
  capabilities, gates, and receipts, not hidden provider calls.
- **Authoring must compress ceremony**: strong primitives are not enough if the
  first real app takes too much wiring to understand.
- **Product pressure must be real**: useful apps solve painful workflows, run in
  one process first, and leave buildable threads.
- **Receipts are business artifacts**: incident reports, delivery manifests,
  transfer receipts, and clinical/mission audit trails are core outputs.
- **Research needs graduation**: expert reports become doctrine, read-only
  reports, package facades, or implementation only through explicit gates.
- **Docs-agent process is a prototype**: the collaboration protocol itself is a
  working model for future runtime handoff.
- **Critical domains sharpen language design**: science, robotics, space, and
  medicine expose primitives enterprise apps can ignore until too late.
