## Graduation Rules

A candidate can move from archive pressure to active proposal only when:

1. It names one narrow fixture.
2. It declares the current gate it touches.
3. It states what remains explicitly out of scope.
4. It uses synthetic data unless a human approves otherwise.
5. It defines receipts/failures before success paths.
6. It checks current status/proposals first.
7. It does not widen Gate 3, Ledger, BiHistory runtime, stream/OLAP runtime, or
   production authority without explicit approval.
