# S3-R204 - if_expr Counterfactual Audit / Branch Intention Boundary

Status: planned
Date: 2026-05-29

Dispatch: R204 = C1-D + C2-P1 -> C3-X -> C4-A -> C5-S

Pattern:
- C1-D designs a narrow counterfactual-audit / branch-intention boundary for
  `if_expr`.
- C2-P1 evaluates whether PROP-032 `assumptions {}` is the right native capsule
  for branch intentions, or whether branch intentions need a distinct surface.
- C3-X pressure-checks the design and assumptions fit for authority drift.
- C4-A accepts, conditions, holds, or redirects the boundary and chooses the
  exact next route.
- C5-S curates the result and current status.

Round intent:
- Preserve runtime laziness: do not evaluate non-selected branches.
- Establish whether the language may know and explain the intentions of
  non-selected branches.
- Keep counterfactual audit as design-only in this round.
- Treat `assumptions {}` as a candidate native epistemic capsule for branch
  intention, not as an already-decided implementation surface.
- Avoid turning branch explanation into dependency/cache authority, runtime
  execution, public API/CLI, release/demo claim, or production behavior.

User framing to preserve:

```text
We should not evaluate non-selected branches, but we should know what contracts
and intentions exist in those branches. Start narrowly: prove the concept that
the language knows branch intentions and can explain them. Check whether
assumptions are the native capsule for branches.
```

Current accepted state:

```text
R203:
  RuntimeSmoke has proof-context consumer evidence for if_expr through the
  existing proof RuntimeMachine path.
  RS-IF1..RS-IF16 PASS, 53/53.
  This is not public runtime support and not production runtime support.
  Counterfactual audit remains future pressure only.

PROP-032:
  assumptions {} is proposed as epistemic provenance:
  named, typed, traceable premises relied on by contracts.
  Assumptions are compile-time declarations, not runtime injection.
```

Global constraints:
- Do not implement code in this round.
- Do not evaluate non-selected branches.
- Do not authorize eager latent-branch execution.
- Do not authorize live runtime changes.
- Do not authorize RuntimeSmoke changes.
- Do not authorize proof RuntimeMachine changes.
- Do not authorize parser, classifier, TypeChecker, SemanticIR emitter,
  assembler, CompilerOrchestrator, CompilerResult, CompilationReport,
  Diagnostics, loader/report, CompatibilityReport, public API/CLI, release,
  package, Spark, or production changes.
- Do not authorize dependency/cache authority, path-sensitive receipts,
  selected-branch cache keys, dynamic dependency tracking, freshness changes, or
  runtime report fields.
- Do not authorize effect sandboxing, dry-run execution, branch replay,
  non-selected branch side effects, or counterfactual execution.
- Do not claim public runtime support, public counterfactual audit support,

  Spark/API/CLI integration.
- Do not mutate accepted R203 evidence.

Terminology for this round:

| Term | Meaning | Status |
| --- | --- | --- |
| Selected branch | The branch evaluated by live runtime under the actual condition. | Already lazy runtime behavior. |
| Non-selected branch | The branch not evaluated by live runtime. | Must remain unevaluated. |
| Branch intention | A static/epistemic explanation of what a branch represents or intends. | Design candidate. |
| Counterfactual audit | The ability to explain what the non-selected branch meant or would represent without executing it. | Future pressure; design-only here. |
| Assumption capsule | A possible PROP-032-aligned container for named premises/intentions attached to branches. | Hypothesis only. |
