# S3-R230 - Resident Supervisor Candidate Intake

Status: Planned
Date: 2026-06-01

## IDD Classification

Mode:

```text
standard / playground-only delegated runtime candidate intake
```

Why:

- R229 accepted the runtime implementation arena and portability boundary;
- resident native supervisor is the next preferred candidate because it
  answers the R228 load-once / execute-many bottleneck;
- the candidate must be intaken before it can affect Main Line routing;
- intake must preserve `igc run`, Reference Runtime, public runtime,
  RuntimeSmoke, stable API, Spark, release, and public performance closure.

Smallest useful artifact:

```text
authorization review -> playground-only proof/intake -> pressure -> acceptance -> status
```

## Round Intent

Decide whether and how the resident native supervisor can become accepted
delegated experimental runtime candidate evidence.

This round should answer:

```text
Can the resident supervisor be intaken as a runtime candidate?
Can it declare a runtime_implementation_id and capability manifest?
Can it prove load-once / execute-many behavior without mainline changes?
Can it preserve lazy branch semantics and Ruby IVM parity?
Can it avoid promoting performance numbers to public claims?
What route should open after intake: artifact passport, temporal backend,
Rust TBackend, todolist app-consumer, or igc run design-only?
```

This round does not implement mainline runtime, CLI, package, report, API,
RuntimeSmoke, Reference Runtime, public support, stable API, Spark integration,
release work, public performance claims, or alternative certification.
