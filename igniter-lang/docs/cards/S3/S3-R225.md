# S3-R225 - Playground Compiler To IVM Adapter Proof

Status: Planned
Date: 2026-05-31

## IDD Classification

Mode:

```text
standard / playground-only runtime adapter authorization before proof
```

Why:

- R223 accepted real executable evidence through the proof RuntimeMachine:
  `.ig -> compile -> .igapp -> delegated experimental runtime -> sum = 42`;
- R224 accepted the IVM playground as sandbox-only delegated experimental
  runtime candidate evidence;
- IVM does not yet consume compiler-emitted `.igapp` /
  `semantic_ir_program.json`;
- the next runtime-productization question is adapter fit, not packaging,
  `igc run`, RuntimeSmoke, or Reference Runtime.

Smallest useful artifact:

```text
authorization review -> playground proof -> pressure -> acceptance -> status
```

## Round Intent

Decide whether a playground-only proof may map compiler-emitted `.igapp` /
`semantic_ir_program.json` into the playground IVM AST/bytecode path and
execute it through IVM.

This round should answer:

```text
Can current compiler output feed IVM without touching mainline runtime surfaces?
Which SemanticIR/operators are adapter-ready?
Which gaps block direct IVM execution?
Does IVM adapter proof preserve lazy branch semantics if a compatible fixture
exists?
Does IVM remain delegated experimental evidence only?
```

This round does not implement runtime productization, helper extraction,
package exposure, `igc run`, RuntimeSmoke changes, Reference Runtime, public
runtime support, Spark integration, or release work.
