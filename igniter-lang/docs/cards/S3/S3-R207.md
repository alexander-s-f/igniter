Status: planned

  forbidden vocabulary misuse.
- C3-A accepts, conditionally accepts, holds, or redirects the docs sync.
- C4-S curates the accepted/held result and exact next Main Line route.

Round intent:
- Make the accepted Level 1 branch-intention vocabulary discoverable in status /
  governance / spec-index surfaces.
- Preserve the R206 target decision: first sync only to low-authority
  discoverability surfaces, not spec-body chapters.
- Keep the core philosophy visible:

```text
Runtime is lazy.
Audit is aware.
```

Current accepted state:

```text
R205:
  Proof-local Level 1 branch-intention concept evidence accepted.
  `if_expr` branch intentions can be statically described for actual and latent
  branches without evaluating latent branches.

R206:
  Level 1 vocabulary accepted as docs vocabulary only.
  C4-A chooses Option A for first docs sync:
    current-status + semantic-governance heat map + optional spec README index.
  `if_expr_branch_intention` remains proof-local / non-canonical.
  Ch2/Ch5/Ch6/Ch7 body edits remain held.
```

Global constraints:
- Do not edit code.
- Do not edit parser/grammar/source syntax.
- Do not edit TypeChecker/SemanticIR/runtime/evaluator/RuntimeSmoke/proof
  RuntimeMachine.
- Do not edit `language-spec.md`, Ch2/Ch5/Ch6/Ch7 body chapters, PROP-032,
  public API/CLI docs, release docs, runtime/report/receipt/CompatibilityReport
  docs.
- Do not authorize live implementation.
- Do not authorize non-selected branch evaluation.
- Do not authorize Level 2 counterfactual dry-run.
- Do not authorize Level 3 comparison report.
- Do not authorize report/result/receipt/CompatibilityReport shape changes.
- Do not authorize dependency/cache authority.
- Do not authorize public counterfactual audit support, public runtime support,
  demo/release/stable/production/all-grammar claims, Spark integration, API/CLI,
  release execution, or production behavior.

Allowed vocabulary:

```text
branch_intention
actual_branch
latent_branch
branch_role
branch_label
condition_observation
static_branch_metadata
intention_source
explanatory_only
non_execution_guarantee
```

Forbidden positive Level 1 claims / output-field vocabulary:

```text
would_result
would_output
would_fail
counterfactual result
counterfactual output
counterfactual failure
latent runtime value
latent runtime failure
latent execution
latent branch execution
simulated branch result
dry-run result
branch replay
replayed branch value
```

Required non-equivalences:

```text
explanatory_only descriptor != runtime execution
branch_intention proof != public counterfactual support
assumptions_shaped_metadata != PROP-032 grammar extension
Level 1 static audit != Level 2 counterfactual dry-run
```
