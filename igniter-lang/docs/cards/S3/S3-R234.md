# S3-R234 - Experimental igc run Slice 0 Authorization

Status: Planned
Date: 2026-06-02

## IDD Classification

Mode:

```text
standard / bounded CLI implementation authorization / experimental executable use
```

Why:

- R233 accepted the experimental `igc run` design-only boundary;
- R233 explicitly opened a later implementation-authorization review, not
  implementation;
- the alpha compiler can produce `.igapp`, and R223/R232 provide executable
  and passport evidence;
- market-window pressure now favors the smallest honest executable CLI loop;
- authority risk remains high enough that implementation must pass through a
  narrow authorization card.

Smallest useful artifact:

```text
authorization review -> bounded implementation -> pressure -> acceptance -> status
```

## Round Intent

Decide whether and how a pre-v1 experimental `igc run` Slice 0 implementation
may begin, then prove or reject that implementation.

Accepted target shape from R233:

```text
igc run ARTIFACT.igapp \
  --passport ARTIFACT.passport.json \
  --input INPUT.json \
  --runtime delegated-experimental:ivm-proof \
  --out RESULT.json \
  --experimental
```

This round should answer:

```text
Can Slice 0 implement a real executable CLI loop?
Can it stay .igapp-only?
Can it validate proof-local passports fail-closed?
Can it keep RuntimeSmoke fully out of the run path?
Can it resolve delegated-experimental:ivm-proof explicitly?
Can it preserve compile CLI compatibility?
Can it avoid stable API, public runtime, production, Spark, release, and
performance claims?
```

This round must not authorize `.igbin` execution, compiler passport emission,
RuntimeSmoke productization, Reference Runtime implementation, public runtime
support, stable API, production readiness, Spark integration, release
execution, public docs claims, or public performance claims.

```text
  adapter path. Runtime selector resolution must not remain implicit.
```
