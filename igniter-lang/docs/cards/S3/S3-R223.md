# S3-R223 - Experimental Executable Quickstart

Status: Planned
Date: 2026-05-31

## IDD Classification

Mode:

```text
standard / authorization-first / experimental executable runtime path
```

Why:

- R222 accepted the experimental-use route and sharpened it to executable
  quickstart, not compile-only quickstart;
- market-window pressure now favors a tiny end-to-end runnable experience;
- runtime movement should happen through the delegated experimental runtime
  layer, without claiming Reference Runtime or production runtime support;
- proof quality remains required, but the proof must be calibrated to a small
  examples-first slice.

Smallest useful artifact:

```text
authorization review -> implementation/proof -> pressure -> acceptance ->
status curation
```

## Round Intent

Authorize and, if authorized, implement the smallest honest experimental
executable quickstart:

```text
.ig source -> compile -> .igapp -> delegated experimental runtime harness
```

The round must move toward runtime usage while preserving the three-runtime
boundary:

```text
Runtime Specification: canonical/normative target, no implementation opened.
Reference Runtime: future canonical implementation candidate, remains closed.
Delegated Experimental Runtime: fast non-canonical harness, allowed only inside
the bounded quickstart if authorized.
```

Compile-only is not success for this round unless executable proof is blocked by
a concrete artifact-format blocker, in which case the correct outcome is HOLD
with an exact blocker.

## Binding Context

R222 accepted:

```text
experimental executable quickstart
three-runtime boundary
implementation still closed until S3-R223-C1-A
stable API / production / public demo / Spark / release claims closed
```

Default candidate implementation write scope, if authorized:
