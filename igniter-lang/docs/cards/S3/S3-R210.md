# S3-R210 - Level 2 Counterfactual Source/Evidence Boundary Design

Status: planned
Date: 2026-05-30

Dispatch: R210 = C1-D -> C2-P1 -> C3-X -> C4-A -> C5-S

Pattern:
- C1-D designs the source/evidence boundary for Level 2 dry-run projections.
- C2-P1 surveys current compiler/SemanticIR/execution-summary evidence surfaces.
- C3-X pressure-checks the boundary for authority drift and closed surfaces.
- C4-A accepts, conditions, holds, or redirects the boundary and chooses the
  next route.
- C5-S curates the result and exact next dispatch.

Round intent:
- Move from R209 hand-authored proof fixtures toward a design boundary for
  emitted/compiler or execution-summary-backed source evidence.
- Decide what can legitimately serve as `source_branch_intention_ref`,
  `input_snapshot_ref`, and `premise_set` source for future proof-local Level 2
  counterfactual dry-run projections.
- Preserve the R209 principle that projection evidence has no runtime,
  report/result/receipt, dependency/cache, public API/CLI, Spark, or production
  authority.

Governing principle:

```text
Runtime is lazy.
Audit is aware.
Dry-run, if ever accepted, must be isolated.
Evidence must be sourced before it can be explained.
```

Current accepted state:

```text
R209:
  Proof-local Level 2 counterfactual dry-run concept evidence is accepted.
  Latent branches can be evaluated inside an experiment-local isolated
  projection envelope with explicit premise_set and no-authority disclaimers.
  Generated output may be called only proof-local Level 2 counterfactual
  dry-run concept evidence.

  Live runtime remains lazy.
  Non-selected branches are not evaluated in live runtime.
  report/result/receipt/cache/API/Spark/public authority remains closed.

Next allowed route:
  Design-only source/evidence boundary.
```

Global constraints:

- Do not edit `.igapp` artifact schema or goldens.
- Do not authorize live runtime non-selected branch evaluation.
- Do not authorize effect execution, external IO, persistence, Ledger/TBackend
  live reads/writes, or non-refusal `tbackend_read`.
- Do not authorize dependency/cache authority.
- Do not authorize public API/CLI, public runtime/counterfactual/demo/stable/
  production/all-grammar claims, release execution, Spark/API/CLI, or production
  behavior.
- This round is design/survey/pressure/decision only.
