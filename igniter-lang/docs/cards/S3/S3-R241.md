# S3-R241 - Experimental igc run Slice 1 VM Candidate Design Boundary

Status: Planned
Date: 2026-06-03

## IDD Classification

Mode:

```text
standard / design-only runtime-productization boundary + facts + pressure + decision
```

Why:

- R240 accepted `igniter-vm` as proof-local delegated experimental VM candidate
  evidence only;
- R240 explicitly opened R241 as a design-only `igc run` Slice 1 boundary;
- the next useful move is to decide how an experimental command may name and
  validate proof-local VM candidate evidence without becoming public runtime
  support;
- implementation, public/runtime/reference/stable/performance/portability
  authority remains closed.

Smallest useful artifact:

```text
Design boundary + surface facts packet + pressure verdict + acceptance decision
```

## Round Intent

Design a pre-v1 experimental `igc run` Slice 1 boundary that may name the
accepted proof-local `igniter-vm` candidate as delegated experimental evidence,
while preserving the strict authority boundary.

This round must decide whether Slice 1 is design-ready for a later bounded
implementation-authorization review, or whether it must redirect first to
Runtime Specification input, VM capability/passport hardening, `.igbin`
output-contract design, or another prerequisite route.

This round must not authorize implementation, `igc run` widening, `.igbin`
execution, compiler passport emission, RuntimeSmoke productization, public
runtime support, Reference Runtime support, stable API, production readiness,
Spark integration, release execution, public performance claims, official/
reference status, alternative certification, or portability guarantees.
