# S3-R242 - Experimental igc run Slice 1 VM Capability Passport Hardening

Status: Planned
Date: 2026-06-03

## IDD Classification

Mode:

```text
standard / proof-local hardening authorization + proof + pressure + decision
```

Why:

- R241 accepted the experimental `igc run` Slice 1 VM candidate design boundary;
- implementation authorization remains held;
- the next blocker is the mismatch between the existing `Add.igapp` passport
  target and the accepted R240 Rust VM candidate `runtime_implementation_id`;
- the smallest useful next artifact is proof-local binding metadata plus a
  fail-closed capability/passport matrix, not CLI/runtime code.

Smallest useful artifact:

```text
Authorization review + proof-local hardening packet + pressure verdict +
acceptance decision
```

## Round Intent

Decide whether a bounded proof-local VM capability/passport hardening proof may
begin for experimental `igc run` Slice 1, then accept or redirect the resulting
evidence.

The proof should bind an `.igapp` artifact to:

```text
runtime_implementation_id:
  igniter.delegated.experimental.vm.rust-tokio.v0

CLI selector:
  delegated-experimental:igniter-vm-candidate
```

The proof must preserve the distinction:

```text
CLI selector: user-facing / experimental / pre-v1
runtime_implementation_id: evidence-facing metadata only
```

This round must not authorize implementation, `igc run` widening, `.igbin`
execution, compiler passport emission, RuntimeSmoke productization, public
runtime support, Reference Runtime support, stable API, production readiness,
Spark integration, release execution, public performance claims, official/
reference status, alternative certification, portability guarantees, loop/
recursion support, or reactive/tbackend daemon execution.
