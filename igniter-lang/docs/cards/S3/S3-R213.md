# S3-R213 - Source-Backed Level 2 Vocabulary Docs Sync Authorization

Status: Planned
Date: 2026-05-30

## Round Intent

Apply the accepted R212 Option A-min vocabulary boundary as a bounded docs-only
sync, if authorized.

This round is not a runtime, report, API, proof, release, or public-claim
round. Its only intended movement is making the source-backed Level 2
counterfactual dry-run vocabulary discoverable in low-authority internal docs
without changing body spec chapters or live surfaces.

## Accepted Vocabulary Boundary

Allowed compact phrase:

```text
source-backed proof-local Level 2 counterfactual dry-run evidence
```

Allowed short explanation:

```text
proof-local, source-backed Level 2 evidence only; non-canonical; no runtime,
report, receipt, cache, API, CLI, Spark, or public authority
```

## Forbidden Claim Set

The docs sync must not claim or imply:

- `counterfactual audit support`
- `counterfactual audit support landed`
- `runtime counterfactual support`
- `public counterfactual support`
- `counterfactual runtime`
- `runtime can evaluate latent branches`
- `runtime can dry-run latent branches`
- `dry-run result`
- `would output`
- `would result`
- `would fail`
- `SemanticIR emits branch_intention`
- `SemanticIR emits source_branch_intention_ref`
- `source-backed branch intentions are SemanticIR`
- `source_branch_intention_ref is a CompilationReport field`
- `source_branch_intention_ref is a CompilerResult field`
- `dry_run_projection is a CompatibilityReport field`
- `receipt contains counterfactual`
- `public API supports counterfactual dry-run`
- `CLI supports counterfactual dry-run`
- `branch-level uses assumptions`
- `PROP-032 branch syntax`

## Global Closed Surfaces

Do not authorize or perform:

- live implementation;
- `lib/**` edits;
- parser, grammar, source syntax, TypeChecker, SemanticIR, runtime, evaluator,
  RuntimeSmoke, proof RuntimeMachine, or CompilerOrchestrator changes;
- report/result/receipt/CompatibilityReport shape changes;
- `.igapp` schema or golden artifact changes;
- live non-selected branch evaluation;
- dependency/cache authority changes;
- public docs, public demo, release, API/CLI, Spark, production, stable,
  all-grammar, or runtime support claims.
