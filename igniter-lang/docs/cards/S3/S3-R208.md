# S3-R208 - Level 2 Counterfactual Dry-Run Boundary Design

Status: planned
Date: 2026-05-30

Dispatch: R208 = C1-D + C2-P1 -> C3-X -> C4-A -> C5-S

Pattern:
- C1-D designs the Level 2 counterfactual dry-run boundary.
- C2-P1 surveys adjacent language/tooling concepts and names what is analogous
  versus non-analogous.
- C3-X pressure-checks for accidental runtime/report/API/cache/public-claim
  widening.
- C4-A accepts, conditions, holds, or redirects and chooses the next exact
  Main Line route.
- C5-S curates the result and current status.

Round intent:
- Move from Level 1 static branch-intention vocabulary toward a design-only
  boundary for Level 2 counterfactual dry-run.
- Define what it could mean for Igniter-Lang to answer "what would have
  happened" for a latent branch without mutating actual runtime results,
  reports, receipts, cache/dependency authority, production behavior, or public
  claims.
- Preserve the distinction:

```text
Level 1: describe branch intention without evaluating latent branches.
Level 2: design the boundary for a possible isolated what-if dry-run.
```

Current accepted state:

```text
R205:
  Proof-local Level 1 branch-intention concept evidence accepted.
  `if_expr` branch intentions can be statically described for actual and latent
  branches without evaluating latent branches.

R206:
  Level 1 branch-intention vocabulary accepted as docs vocabulary only.
  `if_expr_branch_intention` remains proof-local / non-canonical.

R207:
  Bounded Option A docs-sync accepted.
  `branch_intention` is discoverable in low-authority docs.
  Level 2 dry-run remains closed and is routed only as design-only next.
```

Governing phrase:

```text
Runtime is lazy.
Audit is aware.
Dry-run, if ever accepted, must be isolated.
```

Global constraints:
- Do not implement code.
- Do not edit code.
- Do not edit parser/grammar/source syntax.
- Do not edit TypeChecker/SemanticIR/runtime/evaluator/RuntimeSmoke/proof
  RuntimeMachine.
- Do not edit report/result/receipt/CompatibilityReport shapes.
- Do not edit Ch2/Ch5/Ch6/Ch7 body chapters unless C4-A later authorizes a
  separate docs-sync route.
- Do not authorize non-selected branch evaluation in live runtime.
- Do not authorize Level 2 proof or implementation in this round.
- Do not authorize dependency/cache authority.
- Do not authorize effect execution, external IO, Ledger/TBackend production,
  BiHistory production, stream/OLAP production, signing, deployment, or
  production behavior.
- Do not authorize public counterfactual/runtime/demo/stable/production/
  all-grammar claims.
- Do not authorize Spark/API/CLI behavior or claims.

Potential adjacent concepts to survey:

```text
symbolic execution
abstract interpretation
static analysis / control-flow graph explanation
type narrowing / flow typing
proof assistants and verification conditions
contract/refinement systems
debugger what-if / time-travel debugging
logic programming / alternative-world queries
database what-if analysis
probabilistic programming counterfactuals
```

Default stance before R208:

```text
No known analogy grants authority.
Analogies may inform vocabulary and risk boundaries only.
```
