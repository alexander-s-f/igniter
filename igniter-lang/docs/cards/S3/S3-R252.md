# S3-R252 - Contract Invocation Forms Type-Directed Dispatch Proof

Status: Planned
Date: 2026-06-05

## IDD Classification

- Round type: proof-local authorization + implementation proof + pressure
  review + acceptance decision + status curation
- Main route: `S3-R252-C1-A`
- Main track:
  `contract-invocation-forms-type-directed-dispatch-proof-authorization-review-v0`
- Purpose: decide whether a bounded proof-local type-directed dispatch proof
  may begin for contract invocation forms, using R250 forms lowering boundary
  evidence while keeping mainline implementation, stable grammar, runtime, and
  public authority closed.

## Round Intent

R250 accepted the contract invocation forms lowering boundary as design-only.
It accepted LAB-FORMS-P4 as lab-frontier preflight evidence and identified
type-directed dispatch as the next blocking semantic proof before any mainline
forms lowering implementation.

R252 must prove or hold the missing type-directed slice:

- expression-level type facts available to form resolution;
- candidate filtering by operand/result type and trait bounds;
- `E-FORM-AMBIG` after type filtering;
- declaration-order tie-breaker rejection;
- `unresolved_form_error` after type filtering;
- primitive pass-through / unresolved trigger separation;
- `no_form` fail-closed after filtering;
- sidecar trace as audit evidence only.

This round must preserve:

- no mainline parser/typechecker/SemanticIR/runtime/API/CLI/package changes;
- no stable grammar or public API;
- no runtime support or VM linker/subroutine-frame support;
- no `.igapp` or `.igbin` execution authority;
- no compiler passport emission;
- no RuntimeSmoke productization;
- no public runtime or Reference Runtime support;
- no stable API, production, Spark, release, public demo, public performance,
  official/reference status, alternative certification, or portability claims;
- no acceptance of lab behavior as canon.
