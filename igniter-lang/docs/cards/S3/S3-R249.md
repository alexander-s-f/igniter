  decision

## Round Intent

R248 conditionally accepted proof-local specification fixture evidence for
loops and recursion. The packet is useful, but it deliberately did not make the
fixture grammar canonical.

R249 must design the next authoring boundary for managed local recursion and
loop classes:

- bounded local loops;
- structural recursion;
- fuel-bounded recursion;
- `decreases fuel`;
- `for` vs `loop`;
- static vs dynamic `max_steps`;
- Postulate 28 loop naming;
- OOF-L / OOF-R registry stance.

The round must also keep the existing split clean:

- PROP-037 owns service liveness and progression descriptors;
- PROP-039+ owns managed local recursion / loop-class extensions;
- lab behavior and proof fixtures remain evidence, not canon.

This round must preserve:

- no implementation authorization;
- no parser/typechecker/SemanticIR/runtime/API/CLI/package changes;
- no `igc run` widening;
- no `.igapp` or `.igbin` execution authority;
- no compiler passport emission;
- no RuntimeSmoke productization;
- no public runtime or Reference Runtime support;
- no stable API, production, Spark, release, public demo, public performance,
  official/reference status, alternative certification, or portability claims.
