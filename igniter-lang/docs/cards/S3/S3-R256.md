R255 accepted the Igniter Lang repository split boundary as design-ready and
migration-held. R256 now performs the next safe step: a dry-run/file-map proof
that classifies current repository surfaces into explicit destination buckets
without copying them into target repositories as accepted migration.

public claims, framework-to-language authority transfer, lab canon, or archive
repo creation.
