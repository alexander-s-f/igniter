# S3-R211 - Level 2 Source-Backed Counterfactual Proof Authorization

Status: planned
Date: 2026-05-30

Dispatch: R211 = C1-A -> C2-I -> C3-X -> C4-A -> C5-S

Pattern:
- C1-A decides whether a bounded proof-local source-backed Level 2 proof may
  begin.
- C2-I implements/proves only if C1-A authorizes.
- C3-X pressure-checks source evidence, digest chain, authority boundaries, and
  closed surfaces.
- C4-A accepts, conditions, holds, or redirects proof closure.
- C5-S curates the result and exact next dispatch.

Round intent:
- Convert the accepted R210 source/evidence boundary into a bounded proof-local
  source-backed Level 2 counterfactual dry-run proof.
- Replace R209 hand-authored-only branch-intention fixtures with proof-local
  source-backed evidence derived from proof-owned SemanticIR/TypeChecker output
  or secondary proof-owned `.igapp` contract JSON.
- Preserve the R209/R210 rule that Level 2 projection evidence has no runtime,
  report/result/receipt, dependency/cache, public API/CLI, Spark, release, or
  production authority.

Governing principle:

```text
Runtime is lazy.
Audit is aware.
Dry-run, if ever accepted, must be isolated.
Evidence must be sourced before it can be explained.
```

Current accepted state:

```text
R210:
  Source/evidence boundary accepted.
  Tier 1 SemanticIR/TypeChecker is the preferred source, but read-only
  structural citation only.
  Tier 2 execution-summary evidence is allowed narrowly as read-only actual-path
  citation.
  Hand-authored fixtures remain concept-proof-only.
  Refs must be proof-local, non-canonical, and digest-addressed as sha256:<hex>.
  assumed_condition_source is required.

Next allowed route:
  Source-backed proof authorization review only.
```

Global constraints:
- Do not authorize live implementation unless C1-A explicitly authorizes only a
  proof-local experiment route.
- Do not edit `lib/**`.
- Do not edit parser/grammar/source syntax.
- Do not edit TypeChecker/SemanticIR/runtime/evaluator/RuntimeSmoke/proof
  RuntimeMachine.
- Do not edit report/result/receipt/CompatibilityReport shapes.
- Do not edit `.igapp` artifact schema or goldens.
- Do not authorize live runtime non-selected branch evaluation.
- Do not authorize effect execution, external IO, persistence, Ledger/TBackend
  live reads/writes, or non-refusal `tbackend_read`.
- Do not authorize dependency/cache authority.
- Do not authorize public API/CLI, public runtime/counterfactual/demo/stable/
  production/all-grammar claims, release execution, Spark/API/CLI, or production
  behavior.

Required source-backed proof invariants, if C2-I is authorized:

```text
source_branch_intention_ref: structured, digest-addressed, canonical:false
input_snapshot_ref: frozen, digest-addressed, no authority
premise_set: explicit, digest-addressed, no authority
assumed_condition_source: required
projection envelope: no authority, isolated, proof-local only
```
