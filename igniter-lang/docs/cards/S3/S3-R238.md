# S3-R238 - Stdlib Candidate Proof Authorization And Acceptance

Status: Planned
Date: 2026-06-02

## IDD Classification

Mode:

```text
standard / proof-local authorization + implementation proof + pressure decision
```

Why:

- R237 conditionally accepted `playgrounds/igniter-lab/igniter-stdlib` as
  stdlib candidate evidence only;
- R237 carried mandatory C-1/C-2/C-3 scope conditions for temporal, verifier,
  and `.ig` signatures;
- the next useful move is not mainline stdlib implementation, but a bounded
  lab-local proof that makes the evidence packet precise enough for later
  `igniter-vm` candidate intake;
- all public/stdlib/runtime/API/package authority remains closed.

Smallest useful artifact:

```text
Proof authorization review + proof-local implementation + pressure verdict +
acceptance decision
```

## Round Intent

Decide whether a proof-local stdlib candidate proof may begin, execute it if
authorized, pressure-review it, and decide whether the resulting evidence is
accepted.

The proof should harden the R237 candidate boundary:

```text
Decimal FFI evidence
OOF-TC5 / OOF-DM2 behavior
verifier-scope exactness
collections internal-Rust-only classification
temporal domain-specific classification
stdlib/*.ig design-pressure classification
runtime_implementation_id / evidence class / non-claims packet
igniter-vm dependency readiness without opening VM intake
```

This round must not authorize mainline stdlib replacement, public stdlib API,
runtime/API/CLI/package changes, `igc run` widening, `.igbin` execution,
compiler passport emission, RuntimeSmoke productization, Reference Runtime
support, public runtime support, stable API, production readiness, Spark
integration, release execution, public performance claims, official/reference
status, alternative certification, or portability guarantees.
