# S3-R198 - if_expr Live Runtime Evaluator Implementation Design

Status: planned
Date: 2026-05-28

Dispatch: R198 = C1-D -> C2-X -> C3-A -> C4-S

Pattern:
- C1-D designs the live runtime/evaluator implementation boundary for accepted
  expression-level `if_expr`.
- C2-X pressure-checks placement, lazy semantics, error/result surface,
  dependency/cache boundaries, and future counterfactual-audit non-drift.
- C3-A accepts, conditionally accepts, holds, or redirects the design and
  decides whether a later implementation-authorization review may open.
- C4-S curates the outcome and exact next Main Line boundary.

Round intent:
- Move from proof-local runtime/evaluator semantics to a live implementation
  design boundary.
- Preserve the accepted lazy runtime semantics:
  condition first, Bool only, selected branch only.
- Keep counterfactual audit as future pressure, not current implementation
  scope.
- Preserve release/public/Spark/API/CLI and implementation boundaries.
