# S3-R205 - if_expr Counterfactual Audit Concept Proof

Status: planned
Date: 2026-05-29

Dispatch: R205 = C1-I -> C2-X -> C3-A -> C4-S

Pattern:
- C1-I builds the authorized proof-local concept evidence for Level 1 static
  branch audit / branch intention.
- C2-X pressure-checks the proof for non-execution, assumptions drift, and
  closed-surface safety.
- C3-A accepts, conditions, holds, or redirects the proof-local concept closure.
- C4-S curates the outcome and exact next Main Line route.

Round intent:
- Prove the narrow concept that `if_expr` branch intentions can be statically
  described for actual and latent branches.
- Preserve runtime laziness: latent branches are never evaluated.
- Use explanatory-only metadata and optional assumptions-shaped premise refs.
- Keep all concept evidence proof-local.
- Do not create grammar, runtime, report, API, public, release, Spark, or
  production authority.

Current accepted boundary:

```text
R204:
  accepted Level 1 static branch audit / branch-intention boundary
  accepted principle: Runtime is lazy. Audit is aware.
  assumptions are leading candidate capsule for branch premises
  assumptions are not the whole branch-intention surface
  SemanticIR remains native structural source for branch shape
  authorized proof-local concept route next
```

Binding phrase:

```text
actual branches may carry runtime evidence because they ran;
latent branches may carry static explanatory metadata because they exist;
latent branches must not be evaluated to explain them.
```

Global constraints:
- Do not edit `lib/`.
- Do not edit parser, classifier, TypeChecker, SemanticIR emitter, assembler,
  compiler, runtime, RuntimeSmoke, proof RuntimeMachine, public API/CLI,
  loader/report, CompatibilityReport, release/package files, Spark, or
  production behavior.
- Do not authorize live implementation.
- Do not authorize grammar/source syntax changes.
- Do not authorize branch-level `uses assumptions` syntax.
- Do not authorize SemanticIR schema/canon mutation.
- Do not evaluate latent branches.
- Do not perform counterfactual dry-run.
- Do not produce latent runtime values, latent runtime failures, side effects,
  temporal/backend reads, runtime readiness, dependency/cache authority, or
  production behavior for latent branches.
- Do not create report/result/receipt/CompatibilityReport fields.
- Do not mutate accepted R203/R204 evidence.
- Do not execute release commands, publish/yank gems, tag/push/sign/deploy, or
  make public/demo/stable/production/all-grammar/runtime/counterfactual claims.

Forbidden Level 1 output-field vocabulary:

```text
would_result
would_output
would_fail
counterfactual_result
latent_runtime_value
latent_runtime_failure
```

These may appear only in explanatory docs as forbidden terms, not as proof
descriptor keys or accepted output fields.

Required branch-intention record invariant:

```json
{
  "explanatory_only": true,
  "authority": {
    "dependency_authority": false,
    "cache_authority": false,
    "runtime_readiness_authority": false,
    "public_claim": false
  }
}
```
