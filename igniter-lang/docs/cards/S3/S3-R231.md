# S3-R231 - Experimental Runtime Artifact Passport Minimum Boundary

Status: Planned
Date: 2026-06-01

## IDD Classification

Mode:

```text
standard / runtime portability boundary design
```

Why:

- R230 accepted the resident native supervisor as delegated experimental
  runtime candidate evidence;
- R229 already accepted artifact passport as future design vocabulary;
- `igc run` remains closed until artifacts can declare minimum identity,
  capability, authority, and non-claim metadata;
- Rust TBackend, acts-as-tbackend, todolist, C temporal backend, and other
  playground surfaces must not become authority by proximity.

Smallest useful artifact:

```text
design -> surface facts -> pressure -> decision -> status
```

## Round Intent

Define the minimum artifact passport boundary needed before experimental
runtime productization can safely approach `igc run` design or additional
runtime/backend candidate intakes.

This round should answer:

```text
What fields must every experimental executable artifact declare?
How should runtime_implementation_id and capability matching work?
What digest/source/SemanticIR/bytecode evidence is required?
How do we label authority without creating public runtime support?
Can delegated runtime outputs remain useful without becoming canonical?
What must be true before any igc run design-only route opens?
How do IVM, resident supervisor, Rust TBackend, and app-consumer surfaces stay
separate?
```

This round does not implement passport emission, `igc run`, runtime execution,
Reference Runtime, RuntimeSmoke productization, Rust TBackend intake,
acts-as-tbackend intake, public docs, release work, or public claims.
