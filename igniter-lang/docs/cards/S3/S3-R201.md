# S3-R201 - if_expr Proof RuntimeMachine Consumer Authorization + Implementation

Status: planned
Date: 2026-05-28

Dispatch: R201 = C1-A -> C2-I -> C3-X -> C4-A -> C5-S

Pattern:
- C1-A decides whether Slice 2 proof RuntimeMachine consumer implementation may
  begin.
- C2-I runs only if C1-A explicitly authorizes implementation.
- C3-X pressure-checks implementation evidence and closed surfaces.
- C4-A accepts, conditionally accepts, holds, or redirects the implementation.
- C5-S curates the outcome and next Main Line boundary.

Round intent:
- Bridge accepted Slice 1 live internal evaluator into the proof RuntimeMachine
  path without opening `RuntimeSmoke`.
- Add a backward-compatible evaluator adapter hook only if authorized.
- Preserve proof RuntimeMachine ownership of `apply`, `field_access`, and
  `tbackend_read`.
- Prove `.igapp` / contract compute `if_expr` execution through proof
  RuntimeMachine consumer path.
- Keep release/public/Spark/API/CLI/counterfactual/cache surfaces closed.

Current accepted state:

```text
R199:
  accepted live internal direct-require evaluator:
    IgniterLang::SemanticIRExpressionEvaluator
    igniter-lang/lib/igniter_lang/semanticir_expression_evaluator.rb
  supported kinds:
    literal, ref, if_expr
  excluded:
    apply, field_access, tbackend_read
  proof:
    LRT-IF1..LRT-IF15 PASS, 68/68

R200:
  accepted Slice 2 proof RuntimeMachine consumer boundary design
  accepted adapter shape:
    evaluator owns lazy if_expr selection
    proof RuntimeMachine keeps apply, field_access, tbackend_read
  accepted future hook form:
    evaluate(expr, values = {}, call_trace: nil, external_evaluator: nil)
  implementation:
    not authorized by R200
```
