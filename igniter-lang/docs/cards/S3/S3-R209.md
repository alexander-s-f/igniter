# S3-R209 - Level 2 Counterfactual Dry-Run Concept Proof Authorization

Status: planned
Date: 2026-05-30

Dispatch: R209 = C1-A -> C2-I -> C3-X -> C4-A -> C5-S

Pattern:
- C1-A decides whether a bounded proof-local Level 2 concept proof may begin.
- C2-I implements only if C1-A authorizes.
- C3-X pressure-checks proof output for isolation, no-authority, vocabulary, and
  closed-surface integrity.
- C4-A accepts, conditions, holds, or redirects the proof closure.
- C5-S curates the result and next route.

Round intent:
- Convert the accepted R208 Level 2 boundary into a tightly bounded proof-local
  concept proof, if and only if the authorization review confirms the proof
  shape is safe.
- Demonstrate an isolated counterfactual dry-run projection under explicit
  premises without mutating actual runtime results, reports, receipts,
  CompatibilityReport, dependency/cache authority, production behavior, public
  claims, or Spark/API/CLI surfaces.

Governing principle:

```text
Runtime is lazy.
Audit is aware.
Dry-run, if ever accepted, must be isolated.
```

Current accepted state:

```text
R208:
  Level 2 counterfactual dry-run is accepted as a design boundary only:
  explicit isolated proof-local projection under an explicit premise set.
  R208 authorizes no proof, no implementation, and no code edits.
  Next allowed route is only proof-local concept proof authorization review.
```

Global constraints:
- Do not authorize or perform live implementation.
- Do not edit `lib/**`.
- Do not edit parser/grammar/source syntax.
- Do not edit TypeChecker/SemanticIR/runtime/evaluator/RuntimeSmoke/proof
  RuntimeMachine.
- Do not edit report/result/receipt/CompatibilityReport shapes.
- Do not edit `.igapp` artifact schema or goldens.
- Do not authorize live runtime non-selected branch evaluation.
- Do not authorize effect execution, external IO, persistence, Ledger/TBackend
  live reads/writes, or non-refusal `tbackend_read`.
- Do not authorize dependency/cache authority.
- Do not authorize public API/CLI, public runtime/counterfactual/demo/stable/
  production/all-grammar claims, release execution, Spark/API/CLI, or production
  behavior.
- Any proof in this round must be experiment-local only.
