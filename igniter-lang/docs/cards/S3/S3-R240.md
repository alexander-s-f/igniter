# S3-R240 - Igniter VM Candidate Proof Authorization And Acceptance

Status: Planned
Date: 2026-06-03

## IDD Classification

Mode:

```text
standard / proof-local authorization + implementation proof + pressure decision
```

Why:

- R239 accepted `playgrounds/igniter-lab/igniter-vm` as delegated experimental
  VM candidate evidence only;
- R239 identified proof gaps that should be resolved before any `igc run`
  Slice 1, Runtime Specification, Reference Runtime, or public runtime route;
- the next useful move is a lab-local proof that emits machine-readable
  metadata and verifies the hard edges of VM candidate behavior;
- all runtime/API/CLI/package/public authority remains closed.

Smallest useful artifact:

```text
Proof authorization review + proof-local implementation + pressure verdict +
acceptance decision
```

## Round Intent

Decide whether a proof-local `igniter-vm` candidate proof may begin, execute it
if authorized, pressure-review it, and decide whether the resulting evidence is
accepted.

The proof should harden the R239 intake boundary:

```text
runtime_implementation_id / evidence_class / authority_status / non_claims
capability surface
Decimal delegation parity with R238 stdlib dependency context
AOT compiler lowering
stack/register execution
selected branch behavior
non-selected branch silence
unsupported selected-path fail-closed behavior
malformed input / unknown opcode behavior
OP_LOAD_AS_OF / observation trace wording
map-reduce aggregate behavior
reactive/tbackend surface kept classified or explicitly skipped
closed-surface scan
```

This round must not authorize runtime/API/CLI/package changes, `igc run`
widening, `.igbin` execution, compiler passport emission, RuntimeSmoke
productization, public runtime support, Reference Runtime support, stable API,
production readiness, Spark integration, release execution, public performance
claims, official/reference status, alternative certification, or portability
guarantees.
