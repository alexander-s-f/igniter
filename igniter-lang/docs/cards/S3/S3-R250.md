# S3-R250 - Contract Invocation Forms Lowering Boundary

Status: Planned
Date: 2026-06-04

## IDD Classification

- Round type: design boundary + facts packet + pressure review + decision
- Main route: `S3-R250-C1-D`
- Main track: `contract-invocation-forms-lowering-and-execution-boundary-v0`
- Purpose: turn the lab-local forms lowering preflight evidence into an
  Architect-owned design boundary for contract invocation forms, type-directed
  dispatch, SemanticIR lowering, and first runtime stance.

## Round Intent

LAB-FORMS-P1/P2/P4 produced useful frontier evidence:

- current lab forms are `sidecar_resolution_only`;
- real lowering must produce explicit `ContractInvocation` / `Call` shape;
- type-directed dispatch is the main blocker;
- `E-FORM-AMBIG` must remain fail-closed;
- `form:` is only a DX sugar candidate;
- the first runtime stance should prefer compile-time graph inlining /
  monomorphization, while VM subroutine linking remains deferred.

R250 must decide the official boundary for those ideas without authorizing
implementation yet. The goal is to make the future implementation route
reviewable, not to let lab enthusiasm become canon by accident.

This round must preserve:

- no implementation authorization;
- no parser/typechecker/SemanticIR/runtime/API/CLI/package changes;
- no public grammar or stable API claims;
- no production, Spark, release, public demo, public performance, official/
  reference status, alternative certification, or portability claims;
- no acceptance of lab behavior as canon.
