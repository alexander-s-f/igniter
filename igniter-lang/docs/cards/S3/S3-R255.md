## Reservation

S3-R255 is reserved for the Igniter Lang repository split boundary and migration
plan.

Intended target repository:

```text
alexander-s-f/igniter-lang
```

The reserved round should plan the separation between:

- **Igniter Lang**: language specification, proposals, compiler, SemanticIR,
  `.igapp`, experiments, tracks, and conformance-facing artifacts.
- **Igniter Ruby Framework**: Ruby framework, packages, Rails/framework examples,
  Ruby runtime integrations, and application-facing docs.

## Why This Slot Is Reserved

The current monorepo shape causes authority drift: agents see Ruby Framework
concepts, package assumptions, and Ruby examples near language canon and start
pulling framework-specific vocabulary into Igniter Lang.

R255 should make that boundary explicit before any physical repo migration.

## Expected Scope

R255 should decide:

- what belongs in the future `igniter-lang` repository;
- what remains in the Igniter Ruby Framework repository;
- which docs become cross-links rather than copied authority;
- how to preserve git/history/audit evidence;
- whether `igniter-lang/**` becomes the new repo root;
- whether `playgrounds/igniter-lab/**` remains frontier/lab evidence or moves
  later through a separate intake;
- whether a dry-run split route should open next;
- whether physical migration / remote push remains closed or opens later.

## Closed Until R255 Decision

This reservation does not authorize:

- repository migration;
- `git subtree split`, `git filter-repo`, remote push, or release execution;
- package rename;
- public claims;
- CI/package changes;
- framework-to-language authority transfer;
