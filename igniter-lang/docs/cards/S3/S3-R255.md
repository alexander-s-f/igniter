# S3-R255 - Igniter Lang Repository Split Boundary

Status: Planned
Date: 2026-06-06

## IDD Classification

- Round type: design boundary + surface facts packet + pressure review +
  acceptance decision + status curation
- Main route: `S3-R255-C1-D`
- Main track: `igniter-lang-repository-split-boundary-and-migration-plan-v0`
- Purpose: define the repository split boundary between Igniter Lang and the
  Igniter Ruby Framework before any physical migration.

## Round Intent

R254 accepted proof-local contract invocation forms SemanticIR lowering
evidence and preserved R255 as the next Main Line route. R255 now opens the
reserved repository split boundary and migration-plan design round.

The target future language repository is:

```text
alexander-s-f/igniter-lang
```

This round should decide the ownership boundary between:

- **Igniter Lang**: language specification, proposals, compiler, SemanticIR,
  `.igapp`, experiments, tracks, conformance-facing artifacts, and language
  governance.
- **Igniter Ruby Framework**: Ruby framework, packages, Rails/framework
  examples, Ruby runtime integrations, app-facing docs, and framework-specific
  application patterns.

The current monorepo shape creates authority drift: agents see Ruby Framework
concepts, Ruby examples, package assumptions, and application-facing docs near
language canon and start pulling framework-specific vocabulary into Igniter
Lang. R255 should make that boundary explicit before any physical repo
migration.

R255 does not authorize repository migration, `git subtree split`,
`git filter-repo`, remote push, package rename, CI/package changes, release
execution, public claims, framework-to-language authority transfer, or lab
behavior as canon.
