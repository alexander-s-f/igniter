# S3-R203 - if_expr RuntimeSmoke Consumer Proof Authorization + Acceptance

Status: planned
Date: 2026-05-28

Dispatch: R203 = C1-A -> C2-I -> C3-X -> C4-A -> C5-S

Pattern:
- C1-A decides whether a bounded proof-owned RuntimeSmoke consumer harness may
  begin.
- C2-I runs only if C1-A explicitly authorizes implementation/proof.
- C3-X pressure-checks the proof for RuntimeSmoke boundary compliance and
  closed surfaces.
- C4-A accepts, conditionally accepts, holds, or redirects the proof closure.
- C5-S curates the outcome and exact next Main Line boundary.

Round intent:
- Prove, in a proof-owned harness, that existing `RuntimeSmoke.run` can exercise
  the accepted proof RuntimeMachine `if_expr` path.
- Keep `runtime_smoke.rb` unchanged.
- Keep `RuntimeSmoke.run` result shape unchanged.
- Keep `RuntimeSmoke.callback` unchanged.
- Keep `RuntimeSmoke.eval_input_for` unchanged.
- Programmatically generate proof-owned `.igapp` artifacts under the experiment
  `out/` directory.
- Preserve the hierarchy:
  `transitive evaluator load != RuntimeSmoke support != public runtime support != production/runtime claim`.
