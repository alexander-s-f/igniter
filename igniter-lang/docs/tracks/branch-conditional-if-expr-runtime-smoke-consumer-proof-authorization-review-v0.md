## RuntimeSmoke Stances

### Result Shape

`RuntimeSmoke.run` result shape must remain unchanged.

Success key set:

```text
load_status
contract_id
evaluate_status
outputs
compatibility_report_status
trusted
```

Failure key set:

```text
load_status
error
trusted
```

Failure value expectations for evaluator/runtime errors:

```text
load_status: "blocked"
trusted: false
```

### Callback

`RuntimeSmoke.callback` behavior must remain unchanged.

C2-I may inspect/source-scan the callback and may optionally exercise the lambda
shape directly, but it must not use `CompilerOrchestrator#compile(...,
runtime_smoke:)` and must not claim compiler result/report callback integration.

### eval_input_for

`RuntimeSmoke.eval_input_for` must remain unchanged.

No `if_expr` special case may be added. The proof harness must provide explicit
`sample_input` for non-`Add` contracts. The existing `Add` special case remains
unchanged.
