## Rationale

R227 accepted native execution parity through Ruby Fiddle:

```text
FFI-1..FFI-16 PASS
Ruby IVM oracle parity for Add, GT, selected branch, non-selected branch
unsupported selected path fails closed
unsupported non-selected path does not fire
malformed ABI input fails closed
benchmark wording is local/informational only
```

R227 also isolated the next research question:

```text
Can the native runner load a proof-local bytecode file directly, rather than
receiving a Ruby-marshaled in-memory instruction array?
```

That is narrow enough for one playground-only proof. It is also useful because
it tests a realistic AOT execution shape without moving IVM code into
`igniter-lang/lib/**`, creating `igc run`, or promoting the delegated runtime
candidate to Reference Runtime.

Sequencing note:

```text
This is an acceleration-depth closure slice. If C2-I succeeds, C4-A should
explicitly choose whether to shift next toward reusable helper / experimental
use, Runtime Specification input, experimental igc run design-only, or more
AOT hardening. Do not silently continue infinite acceleration research.
```

## Bytecode File Format Policy

C2-I may define a proof-local `.igbin` or equivalent bytecode file format.

Required format stance:

```text
proof-local
non-canonical
not public API
not stable
not Reference Runtime format
not release artifact
```

Recommended minimum header:

```text
magic
format_version
instruction_count
instruction_width_or_abi_marker
payload_digest_or_checksum
```

Recommended payload:

```text
flat instruction records compatible with the R227 ABI:
  int32 opcode
  int32 arg
```

If C2-I chooses a simpler shape, it must explain why the simpler shape is
sufficient for this proof and must still fail closed for malformed files.

## File Read Safety Policy

C2-I must prove fail-closed behavior for file safety.

Required safety cases:

```text
bad magic
unsupported version
truncated header
truncated payload
invalid instruction count
invalid opcode
out-of-bounds jump
halt without RET
```

Allowed error surface:

```text
playground-local error code / local exception / summary field
```

Forbidden error surface:

```text
public diagnostic vocabulary
CompilerResult changes
CompilationReport changes
RuntimeSmoke result-shape changes
API/CLI-visible error contract
```

## Native Runner File-Loading Boundary

Authorized proof path:

```text
Ruby IVM bytecode / playground fixtures
  -> proof-local .igbin or equivalent under playground out/
  -> native runner loads bytecode from file path
  -> native result compared with Ruby IVM oracle
  -> proof summary records parity and safety outcomes
```

Allowed implementation shape:

- C source/header edits under `playgrounds/igniter-runtime/**`;
- Ruby proof harness under `playgrounds/igniter-runtime/**`;
- compiled native library/executable under `playgrounds/igniter-runtime/out/**`;
- bytecode files under `playgrounds/igniter-runtime/out/**`;
- Ruby Fiddle bridge inside the proof harness if useful.

Not authorized:

- root require exposure;
- gemspec/package dependency changes;
- mainline runtime loader;
- CLI `igc run`;
- RuntimeSmoke integration;
- Reference Runtime implementation;
- reusable helper extraction.
