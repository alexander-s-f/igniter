## Closure Criteria

### PROP036-CLI-B1 — Standalone Source Artifact Contract

Closed only when all items below exist.

1. A proof-owned standalone JSON artifact at this stable path:

   ```text
   igniter-lang/experiments/minimal_compiler_profile_finalization_proof/out/compiler_profile_source.stage3_proof.json
   ```

2. The artifact top-level JSON value is the source object itself, not a proof
   summary wrapper.

3. The artifact is generated by this named command:

   ```text
   ruby igniter-lang/experiments/minimal_compiler_profile_finalization_proof/minimal_compiler_profile_finalization_proof.rb
   ```

4. The proof summary records:

   ```text
   standalone_artifact_path
   standalone_artifact_valid: true
   standalone_artifact_exact_forbidden_token_hits: 0
   ```

5. A B1 closure track records the artifact contract, validation result,
   generation command, exact forbidden-token scan result, and non-authorizations.

Not closed by:

- Markdown shape examples only;
- embedded examples in `minimal_compiler_profile_finalization_summary.json`;
- local untracked JSON;
- docs/spec shape without the standalone artifact;
- a generator/registry lookup;
- any artifact that requires discovery/defaulting/env/config/sidecar lookup.

### PROP036-CLI-B3 — Path/Parse Refusal Shape

Closed only when the hybrid refusal model is accepted in the implementation
authorization and proven by tests.

Required model:

```text
Before IgniterLang.compile is called:
  CLI-owned usage/path/JSON preflight refusals emit stderr text only,
  exit non-zero,
  write no OUT.compilation_report.json,
  write no OUT.igapp,
  write no profile-source report JSON.

After IgniterLang.compile is called:
  compiler/orchestrator/assembler refusals keep existing behavior:
  stdout compiler_result JSON,
  OUT.compilation_report.json,
  no OUT.igapp,
  non-zero exit.
```

Required preflight refusal cases:

- `--compiler-profile-source` without path;
- path not found;
- path unreadable;
- path is not a regular file;
- invalid JSON;
- JSON value is not an object/hash;
- unsupported extra arguments in the accepted option set.

Required message constraints:

- one stable stderr line per preflight refusal;
- stdout empty for preflight refusals;
- no raw file contents echoed;
- no parser backtrace echoed;
- no bare loader-status/runtime-readiness tokens.

### PROP036-CLI-B6 — Negative-Token Scan Surface

Closed only when the future CLI proof scans every stream/artifact written or
recorded by the CLI proof matrix.

Forbidden exact tokens:

```text
absent_legacy
present_verified
mismatch
malformed
missing_required
runtime_ready
evaluation_ready
gate3_authorized
runtime_authority
production_ready
```

Required scan surface:

| Scenario | Required scan surface |
| --- | --- |
| no-flag legacy success | `OUT.igapp/**/*.json`, stdout compiler_result JSON, proof summary JSON |
| valid `--compiler-profile-source` success | `OUT.igapp/**/*.json`, stdout compiler_result JSON, proof summary JSON |
| usage / missing flag path | stderr text, proof summary JSON |
| profile source path not found | stderr text, proof summary JSON |
| profile source unreadable / not a file | stderr text, proof summary JSON |
| invalid JSON | stderr text, proof summary JSON |
| JSON value not object | stderr text, proof summary JSON |
| invalid `compiler_profile_id_source` object | `OUT.compilation_report.json`, stdout compiler_result JSON, proof summary JSON |
| source parse / source OOF refusal | `OUT.compilation_report.json`, stdout compiler_result JSON, proof summary JSON |

The scan fails if:

- any forbidden token appears as an exact JSON key/scalar or exact stream token;
- CLI preflight adds a new JSON artifact without adding it to the scan list;
- a proof summary omits stdout/stderr/artifact paths for a covered case.

Allowed qualified source-validation strings such as
`compiler_profile_source.id_digest_mismatch` and
`compiler_profile_source.malformed` must be documented as allowed qualified
compiler-profile-source vocabulary, not loader-status vocabulary.

### PROP036-CLI-B7 — Caller-Facing Source-Shape Docs

Closed only when caller-facing Ruby API docs land.

Required closure:

1. A public/caller-facing doc exists at:

   ```text
   igniter-lang/docs/ruby-api.md
   ```

   or another explicitly approved public API/guide path.

2. `igniter-lang/docs/README.md` links to that doc.

3. The public doc includes `IgniterLang.compile` with:

   ```ruby
   compiler_profile_source: nil
   ```

4. The public doc explains the only supported shapes:

   ```text
   nil
   already-finalized compiler_profile_id_source Hash-like object
   ```

5. The public doc lists required finalized source fields:

   - `kind: "compiler_profile_id_source"`
   - `format_version: "0.1.0"`
   - `status: "finalized"`
   - `profile_namespace: "compiler_profile_unified"`
   - `compiler_profile_id`
   - `descriptor_digest`
   - `finalization_payload_digest`
   - `profile_kind`
   - `slot_order`
   - `slot_assignments`
   - `dispatch_migration_authorized: false`
   - `runtime_authority_granted: false`

6. The public doc states nil preserves `legacy_optional` and manifests omit
   `compiler_profile_id`.

7. The public doc states callers must not pass a file path, raw JSON string, raw
   `compiler_profile_id` string, unfinalized descriptor, runtime-authority
   object, or dispatch-migration object.

8. The public doc states non-authorized surfaces remain closed: CLI profile
   flags, path loading, inline JSON parsing, discovery/defaulting/finalization,
   loader/report, CompatibilityReport, golden migration, `.ilk`, receipts,
   signing, dispatch migration, RuntimeMachine, Gate 3, Ledger/TBackend,
   BiHistory, stream/OLAP, cache, and production authority.

Track docs alone do not close B7.

### PROP036-CLI-B8 — Transport-Only Contract Location

Closed only when all items below are true.

1. The public API/guide doc contains transport-only wording equivalent to:

   ```text
   IgniterLang.compile treats compiler_profile_source: as transport-only. It
   forwards the value unchanged to CompilerOrchestrator#compile.

   The facade does not validate, finalize, discover, infer, load, parse,
   normalize, or default compiler profile sources. Validation and refusal are
   owned by the orchestrator/assembler compiler-profile-source path.

   Changing accepted source shapes is a public API contract change. A future
   card that widens orchestrator/assembler validation must explicitly review
   whether the Ruby facade should expose that widened shape to callers.
   ```

2. The public doc or a named dev-contract section states future
   orchestrator/assembler validation widening does not automatically close the
   facade/API review requirement.

3. Source-level visibility is either:

   - landed by an authorized source-comment card near `compiler_profile_source:`
     in `lib/igniter_lang.rb`; or
   - explicitly deferred by a named docs/governance card, with the deferral path
     recorded in the closing evidence.

Silent absence of a source comment does not close B8.
