# Igniter Application Proposals — Non-Technical Audiences

Date: 2026-04-26.
Perspective: product designer / market strategist.
Purpose: proposals для трёх не-IT аудиторий — конечные пользователи, creators/медиа,
бизнес. IT-аудитория покрыта в `application-proposals.md`.

Ключевой принцип этих proposals отличается от IT-версии:

- **Пользователь не знает и не должен знать про Igniter**
- Igniter — это то, что разработчик использует под капотом
- Ценность: приложение работает проактивно, помнит контекст, ведёт через сложные
  процессы, и создаёт осязаемый результат
- Аналогия: WordPress создан на PHP — пользователи не думают о PHP, они думают
  о своих сайтах

## App 1.1: Forma — Персональный AI-тренер

### Слоган

> "Тренер который тебя знает. Работает пока ты спишь. Адаптируется когда жизнь меняется."

### Боль

Люди хотят быть в форме и здоровы. Но между желанием и системой — пропасть.
Приложения для трекинга собирают данные, но не дают осмысленных рекомендаций.
Персональные тренеры дорого стоят и доступны только в назначенное время.
Программы тренировок из YouTube устаревают через 2 недели когда жизнь меняется.

Forma — это тренер, который не спит. Который видит что ты пропустил две тренировки,
знает почему (ты отметил "командировка"), и уже перестроил план. Который
утром в среду напишет: "Сегодня по плану ноги, но вчера ты мало спал — вот
более лёгкий вариант."

### Как это выглядит для пользователя

**Первый запуск — 10 минут**

Forma проводит пользователя через intake wizard:

```
Привет! Я буду твоим тренером. Мне нужно тебя узнать.

Сколько тебе лет? 34
Какова твоя главная цель? ○ Похудеть  ● Набрать форму  ○ Здоровье
Сколько дней в неделю готов тренироваться? ○ 2  ● 3-4  ○ 5+
Есть ограничения? Спина (L4-L5 грыжа)
Что уже пробовал? Тренажёрный зал 2 года назад, бросил

Проанализирую и составлю план. [Начать]
```

Через минуту — персональная программа, объяснение каждого выбора
("исключила становую из-за спины, заменила на Romanian deadlift — безопаснее
при грыже"), и расписание на месяц.

**Ежедневная жизнь**

Утром Forma присылает уведомление: "Сегодня тренировка A (плечи + кор).
45 минут. Открыть?"

Во время тренировки — пошаговый гид с таймерами, видео техники, адаптацией
в реальном времени ("отдых 90 секунд у тебя обычно лучше, чем 60").

После — короткий check-in: как ощущения? что болит?

**Проактивные сигналы**

```
Forma замечает:
→ Ты пропустил 3 тренировки за 2 недели (обычно — 1)
→ Твои отметки "усталость" участились

"Похоже, нагрузка слишком высокая или что-то происходит.
 Хочешь перейти на поддерживающий режим на 2 недели?"
[Да, снизь нагрузку] [Расскажу что происходит] [Всё норм, продолжаем]
```

**Прогресс через время**

```
Месяц 3 с Forma
────────────────────────────
Тренировок: 36 из 40 запланированных (90%)
Пропусков по болезни: 3
Средний вес: -2.4 кг
Сила: жим сидя +15 кг

Твои личные рекорды этого месяца:
  → Планка: 2:15 (было 0:45 в начале)
  → Отжимания: 24 раза (было 9)

[Посмотреть подробно] [Поделиться]
```

### Igniter под капотом

- **Contracts graph**: расчёт плана как dependency graph (цели → доступные дни →
  ограничения → прогрессия → weekly plan)
- **Proactive agent**: мониторит паттерны, инициирует adaptive adjustments
- **Long-lived session**: вся история тренировок, контекст, прогрессия — живёт
  месяцами
- **Wizard flow**: intake, каждая тренировка, monthly review
- **Compile-time validation**: противоречивые параметры (грыжа + тяжёлые упражнения)
  обнаруживаются до создания плана

### Build Further — что разработчик может добавить

```
Базовый Forma                          Расширения
────────────────────────────────────── ────────────────────────────────────
Встроенные упражнения                → Интеграция со своей базой упражнений
Один пользователь                    → Мульти-семейный аккаунт
Ручной ввод данных                   → Apple Health / Garmin / Fitbit sync
Общие рекомендации по питанию        → Персональный калькулятор КБЖУ
                                     → Интеграция с Cronometer/MyFitnessPal
Еженедельные сводки                  → Интеграция с врачом / нутрициологом
Стандартные программы                → Свои программы (тренер создаёт)
Один язык                            → Многоязычность
```

Capsule model Igniter позволяет каждое расширение поставить как отдельный пакет,
не трогая ядро.

## App 1.2: Meridian — Тьютор для Освоения Навыков

### Слоган

> "Учишься тогда, когда можешь. Система помнит где ты остановился и почему."

### Боль

Онлайн-курсы показывают 90% dropout rate — не потому что люди ленивые, а потому
что курсы не адаптируются к жизни. Начал курс по Python → неделя без занятий
из-за работы → потерял нить → бросил. Флэшкарты скучны. Традиционные учебники
не объясняют ради чего.

Meridian — это тьютор, который понимает твой контекст, объясняет в терминах
твоей жизни, и умеет ждать. Ждать — это суперсила которой нет ни у одного
другого инструмента обучения.

### Как это выглядит для пользователя

**Выбор направления**

```
Что хочешь освоить?

[Язык]  [Программирование]  [Профессия]  [Хобби]  [Своё]

Выбрал: Испанский язык
Цель: путешествие через 4 месяца, выживание + базовое общение
Сколько времени в день: 15-20 минут

Meridian составит план и объяснит почему именно так.
```

**Адаптивная сессия**

Meridian не показывает следующее видео в очереди. Каждая 15-минутная сессия —
диалог. Тьютор выбирает что изучать сегодня исходя из:

- Что ты забыл с прошлого раза (spaced repetition без flash cards)
- Что нужно именно для твоей цели (путешествие → транспорт, еда, ориентация)
- Твоё текущее состояние (сегодня "устал" → лёгкий повтор, не новый материал)

```
Meridian: "Три дня назад мы разбирали числа. Поиграем?
          Я называю цену — ты записываешь сколько это стоит в евро."

Пользователь: cuarenta y dos

Meridian: "Верно! А теперь — в магазине тебе дают сдачу и говорят
          'son tres euros con veinte'. Сколько это?"
```

**Ждать — это feature, не баг**

```
После 18 дней без занятий:

Meridian: "Привет! Прошло 18 дней. Без оценок и упрёков.
          Просто — твоя поездка через 3 месяца 11 дней.
          Хочешь вернуться? Я напомню где мы остановились
          и мы начнём с лёгкого."

[Да, начинаем] [Перенести цель] [Изменить интенсивность]
```

**Monthly story — отчёт как нарратив**

```
Апрельская история Meridian
────────────────────────────
Ты занимался 19 из 30 дней.
Освоил: 143 новых слова, 8 грамматических конструкций.
Самый сложный момент: субхунтатив (потребовал 4 сессии).
Самое быстрое: числа и цены — схватил за одну сессию.

До поездки: 89 дней.
Прогноз Meridian: при текущем темпе — комфортное туристическое общение.
Если добавить 5 мин/день — разговорный уровень.

[Добавить 5 минут] [Оставить как есть] [Рассказать другу]
```

### Build Further

```
Базовый Meridian                       Расширения
────────────────────────────────────── ────────────────────────────────────
Языки (встроенный контент)           → Своя область знания / курс
Текстовые диалоги                    → Голосовые сессии (speech recognition)
Один учащийся                        → Семья / группа / класс
Стандартные цели                     → Корпоративное обучение (команда)
                                     → Преподаватель создаёт курс
                                     → Интеграция с LMS (Moodle, Canvas)
Уведомления в приложении             → WhatsApp / Telegram бот режим
Самооценка                           → Живой тьютор как check-in (раз/неделю)
```

## App 2.1: Studio — Контент-Командный Центр

### Слоган

> "От идеи до результата — один процесс. AI делает тяжёлую работу. Ты принимаешь решения."

### Боль

Создатель контента живёт в режиме permanent chaos: идеи в Notion, черновики в
Docs, публикации в Buffer, аналитика в Excel, коммуникация в Slack. Каждый
пост — ручная сборка из 6 инструментов. AI-инструменты есть, но каждый делает
один шаг — Claude пишет, Canva делает картинки, Buffer публикует. Склейка — ручная.

Studio — это процесс, а не инструмент. Идея входит, контент выходит. Всё
между ними — управляемый, AI-ассистированный конвейер с точками решений.

### Как это выглядит для пользователя

**Intake идеи**

```
НОВАЯ ИДЕЯ
──────────────────────────────────────────
"Написать про то как я перестал читать книги и начал слушать
 конспекты — и почему через 6 месяцев вернулся к обычному чтению"

Studio анализирует:
  Тип: личная история + практика
  Потенциал: высокий (эмоциональный крючок, практический поворот)
  Похожий контент: ваш пост "Как я избавился от notifications" набрал 847 share
  Рекомендуемые форматы:
    ● Newsletter (основная) — длинная форма, личный тон
    ○ Twitter thread — 12 твитов, структура история→инсайт
    ○ LinkedIn — адаптация под профессиональную аудиторию

[Выбрать форматы] [Редактировать идею] [В очередь]
```

**Research Step**

```
Studio собирает контекст:

  ✓ Твои прошлые посты на тему чтения (найдено: 3)
  ✓ Статистика: retention audio vs text (найдено: 2 исследования)
  ✓ Похожий опыт у других авторов (найдено: 5 референсов)
  ✓ Комментарии твоей аудитории на тему (из прошлых постов)

Ключевые инсайты:
  • Твоя аудитория делится 30/70 (аудио-lovers / читатели)
  • Самый частый комментарий: "скорость vs глубина"
  • Исследование MIT: comprehension -23% при аудио vs text

[Посмотреть всё] [Продолжить к структуре]
```

**Структура как checkpoint**

```
СТРУКТУРА DRAFT
────────────────────────────────────────
Hook: "6 месяцев я слушал книги вместо чтения. Потом остановился."
(AI объясняет: сильный открытый крючок, вызывает вопрос "почему?")

Секция 1: Почему перешёл на аудио (контекст + обоснование)
Секция 2: Первые 3 месяца — эйфория производительности
Секция 3: Что начало идти не так (поворот)
Секция 4: Решение вернуться + что понял
CTA: Опрос — а ты?

[Одобрить структуру] [Изменить порядок] [Добавить секцию]
```

После одобрения → Studio пишет черновик. Не публикует. Показывает.

**Review & Edit**

```
ЧЕРНОВИК — Newsletter #47
──────────────────────────────────────────────────────
[Черновик текста с inline AI комментариями]

AI замечает:
  ⚡ Секция 2 длинная — 340 слов (твой обычный средний: 180 слов на секцию)
  ⚡ "исследование MIT" — надо добавить ссылку
  💡 Добавить? Личная цитата из записной книжки — усилит секцию 3

[Редактировать] [Принять все изменения AI] [Одобрить к публикации]
```

**Distribution Planning**

```
ПЛАН ПУБЛИКАЦИИ
────────────────────────────────────────────────────────
Newsletter:  Среда 08:00 (твой исторически лучший open rate день)

Twitter thread: Среда 10:30 (через 2.5ч после newsletter — твои паттерны)
  → Studio адаптировала автоматически (12 твитов, другой тон)

LinkedIn:    Пятница 09:00
  → Studio адаптировала под профессиональный тон

[Посмотреть все версии] [Изменить расписание] [Запустить]
```

**7-дневный отчёт**

```
РЕЗУЛЬТАТЫ — Newsletter #47
────────────────────────────────────────────
Open rate: 34.2% (+4.1pp vs твой средний)
Клики: 847 (лучший результат за 3 месяца)
Ответы на письмо: 23 (обычно: 6-8)
Twitter: 1,247 impressions, 89 retweets

Что сработало:
  → Открытый крючок показал лучший CTR в subject line (A/B: +22%)
  → Секция 3 (поворот) — 67% дочитали до конца vs 41% обычно

Следующий пост: хочешь продолжить тему? (аудитория просила "а что именно
                вернуло к обычному чтению?")
```

### Build Further

```
Базовый Studio                         Расширения
────────────────────────────────────── ────────────────────────────────────
Newsletter + social                  → Podcast show notes автоматически
Manual idea input                    → Идеи из комментариев / вопросов
Одна платформа email                 → Substack, Ghost, Beehiiv, Convertkit
Текстовый контент                    → AI изображения (DALL-E/Midjourney)
Один автор                           → Команда (редактор + автор)
Постфактум аналитика                 → Predictive scoring перед публикацией
Ручные briefing данные               → Автоматический trends monitoring
                                     → Интеграция с CRM для sales контента
```

## App 2.2: Signal — Аудиторная Разведка и Инсайты Роста

### Слоган

> "Твоя аудитория уже говорит тебе что делать. Signal переводит."

### Боль

У любого creator или маркетолога есть данные. Много данных. Analytics, открытия,
клики, комментарии, опросы, продажи. Но данные — это не инсайты. Смотришь на
цифры и думаешь "ок, открытость выросла на 3%" — но что это значит? Что делать?

Типичный creator принимает решения по интуиции ("кажется эта тема заходит")
или по одному громкому посту ("вот это рвануло, надо ещё такое"). Signal делает
то что аналитик за $5000/месяц — только автоматически и постоянно.

### Как это выглядит

**Command Center**

```
SIGNAL — неделя 17 / 2026
────────────────────────────────────────────────────────────────────
Здоровье аудитории: ████████░░  82/100  → стабильно

3 важных сигнала на этой неделе:

  🔴 ВАЖНО: Unsubscribe rate вырос до 1.8% (норма: 0.4%)
           Паттерн: 73% отписавшихся открывали последние 3 письма
           Signal: похоже — не усталость, а мисматч ожиданий
           [Разобраться]

  🟡 ВОЗМОЖНОСТЬ: Сегмент "новые за 30 дней" показывает CTR x2.3
           Они реагируют на "как я это сделал" контент, не на "что делать"
           [Посмотреть сегмент]

  🟢 РОСТ: "Личные истории" посты — retention 67% vs 31% у других
           Твои 3 лучших поста за год: все личные нарративы
           [Видеть паттерн]
```

**Deep Dive по сегменту**

```
СЕГМЕНТ: "Платящие подписчики — активные"
─────────────────────────────────────────────────────────
Размер: 234 человека (8% аудитории, 71% дохода)

Чем отличаются от остальных:
  → Читают 82% писем (vs 31% у всей базы)
  → Кликают на "практика" контент в 4x чаще
  → Средний tenure: 14 месяцев
  → Источник: 67% пришли через одно письмо (про систему заметок)

Что они хотят больше:
  → Комментарии: 41 упоминание "практических примеров"
  → Опросы: топ-тема — "ваш рабочий процесс"

Recommendation:
  "Серия 'Мой рабочий процесс' — по одному инструменту в 2 недели.
   Прогнозирую retention +12% для этого сегмента."

[Начать серию] [Посмотреть подробности] [Оспорить прогноз]
```

**Growth Experiment Wizard**

```
ЭКСПЕРИМЕНТ #12: Subject line тестирование
──────────────────────────────────────────────────────────
Гипотеза: вопросы в subject line дают выше open rate

Signal предлагает:
  Вариант A: "Почему я бросил слушать аудиокниги"  (твой стиль)
  Вариант B: "Ты тоже это делаешь?"                (вопрос)
  Вариант C: "6 месяцев аудио — чем закончилось"   (интрига)

Отправить: сначала 20% базы, победитель — остальным
Метрика победителя: open rate за 4 часа

[Одобрить эксперимент] [Изменить варианты] [Пропустить]

← 3 дня спустя →

Результат: Вариант A победил (34.2% vs 28.1% и 31.7%)
Инсайт: твоя аудитория реагирует на личный нарратив, не на вопросы.
        Это согласуется с паттерном (см. сигнал #3 прошлой недели).

Signal обновил модель. [Посмотреть]
```

## App 3.1: Blueprint — AI-Native Business Process Platform

### Слоган

> "BPMN эпохи AI: бизнес-процессы как validated dependency graphs
>  с агентами, audit trail, и возможностью объяснить каждое решение."

### Боль

Большой бизнес живёт в процессах. Онбординг клиента, согласование договора,
заявка на отпуск, закупочная процедура — всё это процессы. Сегодня они реализованы
через одно из трёх:

1. **Дорогой enterprise software (SAP, ServiceNow)** — работает, но стоит
   миллионы, настраивается полгода, и требует специалистов

2. **Self-built в Jira/Confluence** — дёшево, но не масштабируется, нет
   validation, нет audit, нет AI

3. **Email + Excel** — у всех есть, все ненавидят

AI не исправляет ни один из этих вариантов. ChatGPT не знает где процесс застрял
и почему. Copilot пишет код, но не управляет согласованием.

Blueprint — это "developer-delivered BPMN" с AI-участниками как полноправными
участниками процесса.

### Почему это другое

Традиционные BPMN tools (Camunda, Activiti, IBM BPM):
- XML-based описание процессов (не программисты не работают с этим)
- Отдельные "AI integration" плагины (добавлены сверху, не нативны)
- Нет compile-time validation (ошибки в схеме — в runtime)
- Дорогие лицензии или тяжёлая Java инфраструктура

Blueprint / Igniter:
- Процессы описываются как Ruby contracts (разработчики знают как)
- AI-агенты — такие же участники как люди и сервисы
- Compile-time validation: неверный процесс не запустится
- Receipt chain: каждый шаг оставляет verifiable свидетельство
- Ruby: zero production deps, запускается где угодно

## App 3.2: Accord — Управление Жизненным Циклом Соглашений

### Слоган

> "Каждое соглашение: от инициации до архива. Под контролем. С AI. Аудируемо."

### Боль

B2B бизнес держится на соглашениях: SLA, NDA, partnership agreements, vendor
contracts, employment contracts. Типичный enterprise управляет сотнями активных
договоров — и большинство из них "управляются" через папку на SharePoint и
Excel-таблицу дат renewal.

Accord — это lifecycle management который понимает смысл документа, не только
его метаданные.

## Как Igniter создаёт "нить" между аудиториями

Разработчик осваивает Igniter на IT-уровне (через `application-proposals.md`).

Получив экспертизу — строит Forma для конечных пользователей. Forma популярна →
разработчик получает репутацию. Команда creators замечает паттерн → нанимает
для Studio. Studio вырастает в Signal. Через год enterprise замечает Studio →
просит что-то похожее для внутренних процессов → появляется Blueprint.

Igniter — это платформа. Приложения — это то что пользователи видят.
Репутация строится снизу вверх: пользователи → creators → enterprise.
